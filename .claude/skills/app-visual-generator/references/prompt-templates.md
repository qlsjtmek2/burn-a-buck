## Prompt Structure

### Basic Formula

```
[Subject] + [Style] + [Colors] + [Mood/Atmosphere] + [Technical Specs]
```

### Component Breakdown

**Subject** - What the icon represents
- Single, clear symbol (e.g., "trash bin", "fitness tracker", "music note")
- Avoid complex scenes or multiple objects

**Style** - Visual approach
- Minimalist, 3D, flat design, gradient, illustration, etc.
- Reference specific aesthetic movements (e.g., "Material Design 3")

**Colors** - Color palette
- Be specific: hex codes, named colors, or "vibrant gradient"
- Consider brand colors if applicable

**Mood/Atmosphere** - Emotional tone
- Playful, professional, energetic, calm, friendly, etc.
- Adjectives that convey feeling

**Technical Specs** - Format requirements
- Dimensions, resolution, background type
- Always include for consistency

## Platform-Specific Adaptations

### Android (Material Design 3)

**Adaptive Icon Considerations:**
```
When creating Android app icons, remember:
- Design with a 66×66 dp safe zone (centered)
- Outer 42 dp may be cropped by device masks
- Provide foreground and background layers
- Consider adding a monochrome version for themed icons (Android 13+)
```

**Adaptive Icon Prompt Addition:**
```
"Design suitable for Android Adaptive Icons with 66×66 dp safe zone.
Main elements centered and clearly visible when cropped to various shapes (circle, squircle, rounded square)."
```

### iOS

**iOS Icon Considerations:**
```
- iOS automatically applies rounded corners
- No need to add shadows or gloss effects
- Ensure icon looks good with and without background
- Test on various iOS wallpapers
```

**iOS Icon Prompt Addition:**
```
"Design suitable for iOS with automatic rounded corners applied.
Clean design that works on both light and dark backgrounds.
No pre-applied shadows or rounded corners in source file."
```

### Cross-Platform

**Universal Prompt Addition:**
```
"Design suitable for both iOS and Android platforms.
Simple, recognizable composition that adapts to various shapes and sizes.
High contrast, works on light and dark backgrounds."
```

## Prompt Customization Workflow

### Step 1: Identify Core Elements
- What does the app do? (Function)
- Who is the target user? (Audience)
- What feeling should it evoke? (Mood)

### Step 2: Choose Style
- Select from: Minimalist, 3D, Flat, Gradient, Playful, Professional
- Consider 2025 trends: Minimalism (31% higher recall), 3D (24% higher engagement), Gradients (28% better visibility)

### Step 3: Define Colors
- Use brand colors if available
- Or select from palette categories above
- Ensure high contrast for visibility

### Step 4: Select Template
- App Icon (512×512): Choose Template 1-5 based on style
- Feature Graphic (1024×500): Choose Template 1-3 based on content focus

### Step 5: Customize Prompt
- Fill in [PLACEHOLDERS] with specific details
- Add platform-specific adaptations if needed
- Include style variations keywords for refinement

### Step 6: Generate & Iterate
- Generate image with AI tool (Midjourney, DALL-E, Stable Diffusion)
- Review against design principles (see design-principles.md)
- Refine prompt and regenerate if needed
- A/B test multiple variations
